# AI 태그 및 한국어 요약 제목 설계

## 목표

`mode=ai` 북마크 등록과 AI 재분류 시 페이지 내용을 한국어 제목 스타일로 요약하고, 한국어 태그 3~5개를 생성한다. 사용자는 태그를 직접 편집할 수 있으며 기존 검색창과 태그 클릭으로 태그가 붙은 북마크를 찾을 수 있다.

## 범위

- AI 분류 시 기존 `title`을 최대 40자의 한국어 요약 제목으로 교체한다.
- AI 분류 시 한국어 태그 3~5개를 생성한다.
- 북마크 카드에 태그를 표시하고 태그 클릭 검색을 제공한다.
- 북마크 편집 화면에서 태그 추가와 삭제를 지원한다.
- 기존 통합 검색이 제목, URL, 설명, 태그를 함께 검색한다.
- 별도 태그 관리 화면, 태그 통계, 다중 태그 전용 필터는 만들지 않는다.
- `manual` 및 `none` 등록에서는 AI 제목과 태그를 생성하지 않는다.

## 데이터 모델

`bookmarks` 테이블에 다음 컬럼을 추가한다.

```sql
tags text[] not null default '{}'
```

태그는 북마크에 종속된 값이므로 별도 `tags` 테이블을 만들지 않는다. 개인용 서비스에서 태그 일괄 이름 변경이나 통계가 요구되지 않으므로 배열 모델이 가장 단순하다. 정확한 태그 필터를 위해 GIN 인덱스를 추가한다.

기존 북마크는 빈 배열을 기본값으로 받아 별도 데이터 변환 없이 호환된다.

## 태그 규칙

- AI 결과: 3~5개
- 사용자 수정 결과: 0~5개
- 태그 하나의 길이: trim 후 1~20자
- 저장 전 앞뒤 공백 제거
- 동일한 문자열은 한 번만 저장
- AI가 생성하는 태그는 한국어로 작성한다. 제품명이나 고유 기술명처럼 번역하면 의미가 훼손되는 고유명사는 원문 표기를 허용한다.
- 태그 앞에 `#`을 저장하지 않는다. UI에서 칩 형태로만 표현한다.

## AI 응답과 처리

기존 카테고리 분류 응답에 다음 공통 필드를 추가한다.

```ts
{
  category: CategorizeResult;
  summaryTitle: string; // 한국어 제목 스타일, 1~40자
  tags: string[];       // 3~5개
}
```

Gemini, Anthropic, OpenAI 모두 각 SDK의 구조화 출력 기능으로 동일한 의미의 응답을 받는다. OpenAI는 SDK 요구에 맞게 루트 객체를 유지한다. 최종 응답은 zod로 검증한다.

처리 순서:

1. 메타데이터에서 원본 제목과 설명을 수집한다.
2. 현재 카테고리와 페이지 메타데이터를 AI에 전달한다.
3. AI가 카테고리 결과, 한국어 요약 제목, 한국어 태그를 한 번에 반환한다.
4. 북마크가 여전히 `ai_status='pending'`인 경우에만 카테고리, `title`, `tags`, `ai_status='done'`을 적용한다.
5. AI 요청 또는 응답 검증이 실패하면 `ai_status='failed'`로 변경하고 기존 제목과 태그는 보존한다.

사용자가 AI 재분류를 명시적으로 실행하면 기존 제목과 태그는 새 AI 결과로 교체된다. 사용자가 편집 화면에서 제목이나 태그를 수정하는 것은 허용하며, 이후 재분류 전까지 유지한다.

## API

### 북마크 응답

모든 북마크 응답에 `tags: string[]`를 포함한다.

### 북마크 수정

`PATCH /api/bookmarks/:id` 요청에 다음 필드를 추가한다.

```json
{ "tags": ["React", "프론트엔드", "웹 개발"] }
```

서버는 최대 개수, 길이, 공백 제거 및 중복 제거를 검증한다. 빈 배열은 모든 태그 삭제를 의미한다. API 경계에서는 `packages/shared`의 zod 스키마를 사용한다.

### 검색

`GET /api/bookmarks?q=...`의 의미를 제목, URL, 설명, 태그 검색으로 확장한다. 사용자 경계를 포함하는 Postgres 검색 함수를 마이그레이션으로 추가하여 배열 내부 문자열도 부분 일치 검색한다. 함수는 `user_id`, 검색어, 카테고리, 커서, limit을 입력받고 기존 `(created_at, id)` keyset 정렬과 응답 형태를 유지한다.

태그 클릭도 별도 API를 만들지 않고 기존 `q`에 태그 문자열을 전달한다. 따라서 카테고리 필터와 태그 검색을 동시에 적용할 수 있다.

## 웹 UI

- 카드의 요약 제목 아래에 태그 칩을 표시한다.
- 칩은 줄바꿈 가능하며 태그가 없으면 영역을 렌더링하지 않는다.
- 태그 칩 클릭 시 링크 이동을 막고 검색창 값을 해당 태그로 설정한다. 디바운스 후 기존 검색 요청이 실행된다.
- 편집 다이얼로그에 태그 입력을 추가한다.
  - Enter 또는 쉼표로 현재 입력을 태그로 추가한다.
  - 칩의 삭제 버튼으로 태그를 제거한다.
  - 최대 5개에 도달하면 추가 입력을 제한하고 안내 문구를 표시한다.
- 모바일과 데스크톱에서 동일한 정보 구조를 사용한다.

## 오류 처리

- AI 응답이 제목 길이 또는 태그 개수 조건을 위반하면 전체 응답을 실패로 처리하고 부분 저장하지 않는다.
- 수동 태그 요청이 유효하지 않으면 공통 `VALIDATION_ERROR` 400 응답을 반환한다.
- 검색 함수 오류는 기존 공통 에러 미들웨어로 전달한다.
- AI 실패는 북마크 생성 자체를 취소하지 않으며 기존 재시도 UI를 사용한다.
- 15초 AI 타임아웃 및 재시도하지 않는 정책은 기존 명세를 유지한다.

## 테스트와 검증

- 마이그레이션: 컬럼 기본값, GIN 인덱스, 사용자 경계를 포함한 검색 함수
- 공유 스키마: 태그 최대 개수, 태그 길이, 빈 배열, 중복 정규화
- AI provider: 세 provider의 제목·태그 구조화 응답과 잘못된 응답 거부
- 분류 서비스: 성공 시 제목·태그 저장, 실패 시 기존 값 보존, pending 조건부 업데이트
- API: 태그 수정, 삭제, 태그 부분 검색, 카테고리와 검색 조합, 다른 사용자 데이터 차단
- 웹: 카드 태그 렌더링, 태그 클릭 검색, 편집 추가·삭제·최대 개수
- 전체 검증 루프: `bun run typecheck && bun run lint && bun run test && bun run build`
