# 북마크 카드 메뉴 겹침 수정 설계

## 문제

홈의 북마크 목록은 각 행을 `transform`으로 배치하는 가상 목록이다. 각 행이 별도 stacking context를 만들기 때문에 카드 메뉴의 `z-index`가 높아도 다음 행이 나중에 그려지면서 메뉴를 가린다. 현재 행의 `focus-within:z-10`은 마우스·키보드에서는 동작할 수 있지만, 버튼 탭이 포커스를 유지하지 않는 iOS Safari에서는 신뢰할 수 없다.

## 설계

가상 목록 행에 식별용 클래스 `virtual-bookmark-row`를 추가한다. 행 내부의 메뉴 트리거가 `aria-expanded="true"`이면 CSS `:has()` 선택자로 해당 행의 `z-index`를 올린다.

메뉴 상태와 접근성 속성은 기존 `BookmarkCard`가 계속 관리한다. 상위 컴포넌트에 열린 메뉴 ID를 추가하거나 메뉴를 portal로 옮기지 않는다. 따라서 메뉴 열기·닫기·포커스 복원 동작은 바뀌지 않는다.

## 검증

- 컴포넌트 테스트에서 가상 목록 행에 전용 클래스가 있는지 확인한다.
- 스타일 회귀 테스트에서 열린 메뉴를 포함한 행의 stacking 규칙을 확인한다.
- 기존 메뉴 열기·바깥 클릭·Escape·액션 후 닫기 테스트를 함께 실행한다.
- 전체 typecheck, lint, test, build를 통과시킨다.
