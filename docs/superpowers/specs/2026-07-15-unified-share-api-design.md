# iOS 단축어 통합 공유 API 설계

## 목표

iPhone 공유 시트에는 `북마크 저장` 단축어 하나만 노출한다. 단축어는 링크와 이미지를 구분하지 않고 같은 API에 같은 폼 필드로 전송하며, 서버가 입력 종류를 판별해 기존 링크 등록 또는 이미지 등록 흐름으로 전달한다.

기존 웹 클라이언트와 외부 호출의 호환성을 위해 `POST /api/bookmarks`와 `POST /api/images`는 유지한다.

## API 계약

새 엔드포인트는 `POST /api/share`다. Bearer JWT와 기존 범위 제한 API Key 인증을 모두 허용한다.

요청은 `multipart/form-data`이고 필드 이름은 `item`이다.

- `item`이 텍스트 폼 필드이면 URL로 해석한다.
- `item`이 파일 폼 필드이면 이미지로 해석한다.
- 텍스트와 파일이 동시에 들어오거나 둘 다 없으면 `400 VALIDATION_ERROR`를 반환한다.
- URL은 기존 URL 정규화와 중복 검사를 그대로 거치며 `mode=ai`로 등록한다.
- 파일은 기존 이미지 형식, 20MB, 64MP, 실제 시그니처 및 디코딩 검증을 그대로 거친다.
- 성공 응답은 기존 등록 API와 동일한 `201 { bookmark }`다.
- URL 중복은 기존과 동일하게 `409`, 파일 크기 초과는 `413`, 지원하지 않는 이미지 형식은 `415`다.

요청당 항목은 하나만 받는다. 여러 이미지를 공유하면 iOS 단축어의 `각 항목 반복`이 같은 엔드포인트를 이미지마다 호출한다. 링크와 이미지가 섞인 공유 입력도 각 항목을 독립적으로 전송한다.

## 서버 구조

라우트끼리 HTTP 요청을 전달하거나 내부적으로 다른 라우트를 호출하지 않는다. 현재 링크·이미지 라우트에 들어 있는 생성 흐름을 각각 재사용 가능한 서비스 함수로 분리한다.

- 링크 생성 함수: URL 정규화, 카테고리/AI 상태 결정, DB insert, 비동기 메타데이터·AI 작업 시작
- 이미지 생성 함수: 이미지 처리, Storage 저장, DB insert, 실패 정리, 비동기 AI 작업 시작, 썸네일 서명
- 기존 `/api/bookmarks`와 `/api/images`는 각 함수를 호출한다.
- 새 `/api/share`는 multipart 입력 판별만 담당하고 적절한 함수를 호출한다.

이 경계로 웹 업로드, 기존 단축어, 새 통합 단축어가 같은 저장 규칙을 사용하게 한다.

## iOS 단축어

단축어 이름은 `북마크 저장`으로 하고 공유 시트 입력 유형은 URL, Safari 웹페이지, 이미지를 허용한다.

1. `각 항목 반복`으로 공유 입력을 순회한다.
2. 반복문 안에서 `POST https://<서버주소>/api/share`를 호출한다.
3. `X-API-Key` 헤더만 설정한다.
4. 요청 본문을 폼으로 선택하고 `item` 값에 현재 반복 항목을 넣는다.
5. `Content-Type`은 직접 설정하지 않고 단축어가 multipart boundary를 생성하게 한다.
6. 반복 종료 후 성공·실패 건수를 알림으로 표시한다.

서버의 입력 판별 덕분에 단축어 내부에는 URL/이미지 조건 분기가 필요 없다.

## 오류 처리와 보안

- 인증, 사용자 소유권, API Key rate limit은 기존 API와 동일하게 적용한다.
- 파일명이나 MIME 선언만 신뢰하지 않고 기존 이미지 처리기가 실제 파일을 검증한다.
- Storage 저장 후 DB insert 실패 시 원본과 썸네일을 정리한다.
- 로그에 API Key, 이미지 바이트, 서명 URL을 남기지 않는다.
- 부분 성공이 가능하므로 여러 항목 중 한 요청이 실패해도 단축어 반복을 계속하고 마지막에 성공·실패 수를 함께 표시한다.

## 테스트와 검증

- shared/API 경계: URL 텍스트, JPEG 파일, HEIC 파일, 입력 없음, 텍스트와 파일 동시 입력, 잘못된 URL, 비이미지 파일, 크기 초과
- 회귀: `/api/bookmarks`와 `/api/images`가 분리된 서비스 함수를 사용한 뒤에도 기존 응답과 오류를 유지
- 인증: Bearer와 API Key 성공, 무인증 실패, API Key rate limit 범위 유지
- 비동기 처리: 링크 AI/메타데이터와 이미지 AI 작업이 기존처럼 `201` 이후 시작
- 전체 검증: `bun run typecheck && bun run lint && bun run test && bun run build`
- 실기기: iPhone 공유 시트의 단일 단축어로 Safari URL, JPEG, HEIC, 여러 이미지 공유를 각각 확인

## 범위 제외

- 기존 `/api/bookmarks`와 `/api/images` 제거 또는 deprecation
- 서버가 한 HTTP 요청에서 여러 파일을 일괄 처리하는 기능
- 텍스트 메모, 동영상, PDF 등 링크·이미지 외 공유 입력
- 네이티브 iOS Share Extension
