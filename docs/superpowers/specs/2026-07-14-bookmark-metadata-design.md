# 북마크 범용 메타데이터와 네이버지도 검색 링크 설계

## 목표

링크와 이미지에서 식당명·지점·지역을 충분히 확실하게 식별한 경우 네이버지도 검색 결과를 여는 링크를 자동으로 추가한다. 지도 전용 컬럼이나 지도 전용 UI를 만들지 않고, 모든 북마크가 공유하는 문자열 key-value 메타데이터로 저장·조회·편집·표시한다.

메타데이터는 이후 사용자가 다른 부가정보를 직접 추가하는 데도 사용할 수 있다. 이번 범위에서 AI가 자동 생성하는 메타데이터는 네이버지도 검색 링크 하나로 제한한다.

## 범위

### 포함

- `bookmarks`의 범용 key-value 메타데이터 저장
- AI의 확신도 높은 식당·지점·지역 식별
- 서버에서 생성하는 네이버지도 검색 URL
- 카드와 이미지 상세 화면의 공통 메타데이터 표시
- 링크·이미지 편집 화면의 메타데이터 추가·수정·삭제
- 기존 AI 조건부 업데이트와 사용자 편집 우선 규칙 적용

### 제외

- 네이버 장소 API를 이용한 정확한 place id 매칭
- 좌표, 길찾기, 지도 임베드
- 메타데이터 전용 검색·필터·정렬
- AI가 임의의 메타데이터 키 또는 외부 URL을 생성하는 기능
- 인스타그램·스레드 로그인이나 비공개 본문 수집

## 데이터 모델

`bookmarks`에 다음 컬럼을 추가한다.

```sql
metadata jsonb not null default '{}'::jsonb
```

값은 중첩 객체가 아닌 `Record<string, string>`만 허용한다. HTTP 경계의 공유 zod 스키마는 다음 제한을 적용한다.

- 최대 10개
- key: trim 후 1~40자, 빈 key와 중복 key 금지
- value: trim 후 1~2048자
- prototype pollution에 쓰일 수 있는 `__proto__`, `prototype`, `constructor` key 금지
- URL 값은 저장 시에도 문자열이며 `http:` 또는 `https:`일 때만 UI에서 외부 링크로 렌더링

DB에는 JSON object 제약을 추가한다. 항목 개수와 문자열 길이 검증은 API 경계와 AI 적용 경계에서 수행한다. 현재 서비스의 모든 데이터 쓰기가 Express를 거치므로 복잡한 JSON 순회 check function은 추가하지 않는다.

기존 행은 기본값 `{}`를 사용한다. `search_bookmarks` 반환 타입과 목록·상세 DB mapper에 `metadata`를 추가한다. 메타데이터는 이번 범위에서 검색 대상에 포함하지 않는다.

## AI 분석과 지도 링크 생성

AI의 strict 구조화 응답에는 범용 메타데이터 객체를 직접 받지 않는다. 대신 다음 nullable 장소 후보를 추가한다.

```ts
interface PlaceCandidate {
  name: string;
  locality: string | null;
  confidence: number;
}
```

프롬프트는 입력이 식당·카페·주점 등 방문 가능한 음식점이고 상호명을 직접 확인할 수 있을 때만 후보를 반환하도록 한다. 지점명은 `name`에 포함하고, 게시물·캡션·이미지에서 확인되는 동네·도시·주소 단서는 `locality`에 넣는다. 음식 사진만 있고 상호를 알 수 없거나, 여러 매장이 섞였거나, 추측에 의존하면 `null`을 반환한다.

서버는 `confidence >= 0.85`이고 정규화된 `name`이 비어 있지 않을 때만 지도 메타데이터를 만든다. 검색어는 `name`과 존재하는 `locality`를 공백으로 합치고 URL 인코딩한다.

```text
key: 네이버지도
value: https://map.naver.com/p/search/{encoded query}
```

AI는 URL을 만들지 않는다. 따라서 모델이 임의 도메인이나 잘못된 URL을 반환할 수 없다. 링크 입력은 기존 SSRF 보호 메타데이터 fetch 결과와 AI가 보는 제목·설명·사이트명을 사용한다. 인스타그램·스레드처럼 공개 HTML 메타데이터가 제한된 링크는 수집 가능한 정보 안에서만 판별하며, 정보가 부족하면 지도 메타데이터를 생성하지 않는다. 이미지 입력은 기존 최대 2048px 임시 분석 JPEG를 사용한다.

분석 성공 시 기존 `ai_status='pending'` 조건부 UPDATE 안에서 결과를 적용한다. AI가 만든 `네이버지도` key는 기존 메타데이터에 병합하고 같은 key가 있으면 새 분석 결과로 교체한다. 다른 사용자 key는 보존한다. 후보가 없거나 임계값 미만이면 기존 key를 자동 삭제하지 않는다. 사용자가 원하지 않는 링크를 지운 뒤 분석 결과가 늦게 도착하는 경우에는 메타데이터 편집이 `ai_status='idle'`로 전환되어 조건부 UPDATE가 실패하므로 삭제가 보존된다. 사용자가 이후 명시적으로 재분석하면 같은 key가 다시 생성될 수 있음을 확인 문구에 포함한다.

## API

모든 북마크 응답에 `metadata: Record<string, string>`을 포함한다. 기존 행과 잘못된 nullable DB mock은 mapper에서 `{}`로 정규화하되, 실제 쓰기 경계에서는 공유 zod 스키마를 통과한 값만 저장한다.

`PATCH /api/bookmarks/:id`에 선택적 `metadata`를 추가한다. 전달하면 전체 객체를 교체하고, 생략하면 기존 값을 유지한다. 빈 객체는 전체 메타데이터 삭제다. 기존 title·description·categoryId·url·tags 편집과 동일하게 메타데이터 편집도 `ai_status='idle'`로 전환해 진행 중인 AI 결과를 폐기한다.

별도 메타데이터 CRUD endpoint는 추가하지 않는다. 개인용 서비스이고 최대 10개이므로 기존 북마크 편집 mutation 하나가 가장 단순하다.

## 웹 UX

### 카드와 상세

링크 카드와 이미지 카드 모두 제목·요약 아래, 태그 위에 메타데이터를 표시한다. 이미지 상세도 같은 표현을 사용한다.

- URL 값: key를 라벨로 사용하는 외부 링크. `target="_blank"`, `rel="noreferrer"`를 적용한다.
- 일반 문자열: `key: value` 텍스트
- `네이버지도`도 전용 컴포넌트나 전용 버튼이 아니라 위 URL 규칙으로 렌더링한다.
- 카드 전체 링크가 있는 이미지에서는 메타데이터 링크가 더 높은 stacking layer와 pointer events를 가져 카드 상세 이동을 가로채지 않는다.
- 긴 값은 카드에서 한 줄 말줄임하고 상세·편집 화면에서는 전체를 확인할 수 있다.
- 메타데이터가 비어 있으면 기존 레이아웃에 빈 영역을 만들지 않는다.

### 편집

기존 링크·이미지 편집 폼에 `메타데이터` 섹션을 추가한다. 각 행은 key 입력, value 입력, 삭제 버튼으로 구성하며 `항목 추가`로 최대 10개까지 늘릴 수 있다.

- 저장 시 key/value를 trim한다.
- 빈 행은 저장 전에 제거한다.
- key만 또는 value만 입력된 행과 중복 key는 인라인 오류로 저장을 막는다.
- URL 값도 일반 텍스트 입력으로 편집하며 별도 URL 타입 선택은 두지 않는다.
- AI가 생성한 `네이버지도` 행도 동일하게 편집·삭제할 수 있다.

## 오류 처리

- AI 장소 후보가 없거나 임계값 미만: 분석의 나머지 제목·요약·태그·카테고리는 정상 적용하고 메타데이터는 변경하지 않는다.
- 장소 후보 스키마 오류: 전체 AI 응답 경계 오류로 처리해 기존 `failed` 전이를 따른다.
- URL 조립 실패나 길이 제한 초과: 지도 메타데이터만 생략하고 나머지 분석 결과는 적용하며 warning에는 bookmark id와 단계만 기록한다.
- PATCH 메타데이터 검증 실패: 공통 400 오류 포맷을 반환하고 DB를 변경하지 않는다.
- 잘못된 URL 문자열: 저장은 허용하되 일반 텍스트로 렌더링한다. URL로 렌더링하는 판단은 `URL` 파싱 후 `http:`/`https:` protocol 확인으로 제한한다.

## 테스트와 수용 기준

### 자동 테스트

- shared: key/value trim, 최대 개수·길이, 금지 key, 중복 key, 빈 객체
- migration/docs: `metadata jsonb not null default '{}'`와 `search_bookmarks` 반환 포함
- AI: nullable 장소 후보, confidence 경계, 프롬프트의 추정 금지, strict JSON Schema
- API service: 0.85 이상일 때만 서버가 네이버지도 URL을 생성하고 사용자 key를 보존
- API route/mapper: 목록·상세 응답, PATCH 전체 교체, 빈 객체 삭제, pending 결과 차단
- web: 카드와 상세의 URL/텍스트 렌더링, 외부 링크 속성, 이미지 카드 클릭과 메타데이터 링크 충돌 방지
- web form: 행 추가·수정·삭제, 빈/중복/최대 10개 검증, 저장 payload

### 수동 확인

- 공개 인스타그램 또는 스레드 식당 링크에서 상호·지역이 명확할 때 카드와 상세에 `네이버지도` 링크 표시
- 상호가 없는 음식 사진에는 지도 링크가 생기지 않음
- 식당 간판과 지역 단서가 선명한 이미지에는 지도 링크 표시
- 지도 링크가 새 탭에서 올바른 네이버지도 검색 결과를 엶
- 사용자가 `네이버지도`를 삭제하거나 임의 메타데이터를 추가한 뒤 새로고침해도 유지
- 모바일 375px에서 긴 key/value와 편집 행이 가로 overflow를 만들지 않음

전체 `bun run typecheck && bun run lint && bun run test && bun run build`가 통과해야 완료다. 마이그레이션 원격 적용은 구현 커밋과 분리해 사용자의 배포 범위에 맞춰 수행하고, 결과를 `PROGRESS.md`에 기록한다.
