# AI 사용 모델 카드 레이아웃 설계

## 목표

설정의 AI 분류 섹션에서 사용 모델 선택을 API 키 관리보다 먼저 보여주고, 독립된 카드 경계로 두 설정의 역할을 더 명확히 구분한다.

## UI

AI 분류 제목과 설명 아래 순서를 다음으로 변경한다.

1. 사용 모델
2. AI API 키

사용 모델 영역은 `rounded-xl border border-zinc-200 p-4 dark:border-zinc-800` 카드로 감싼다. 카드 내부에는 기존과 동일한 모델 select와 `모델 저장` 버튼을 배치한다. 등록된 provider 키가 없으면 같은 카드 내부에 기존 빈 상태 문구를 표시한다.

API, 상태, mutation, 모델 필터링 동작은 변경하지 않는다. 모바일에서는 세로 배치, `sm` 이상에서는 select와 버튼을 가로 배치하는 기존 반응형 동작을 유지한다.

## 검증

- DOM에서 `사용 모델` 제목이 `AI API 키`보다 앞에 존재
- 사용 모델 영역에 독립 카드 스타일 적용
- 모델 저장과 키 0개 빈 상태 회귀 테스트
- 전체 typecheck/lint/test/build 통과
