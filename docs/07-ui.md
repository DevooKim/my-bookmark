# 07. UI/UX

원칙: **심플·모던, 모바일 퍼스트, 다크모드 지원**. 화면 수를 최소화하고 밀도를 낮게 유지한다.

## 디자인 시스템

- **Tailwind CSS v4** (CSS-first 설정, `@theme`). 별도 tailwind.config 불필요한 v4 방식 사용.
- 컴포넌트: shadcn/ui 스타일(Radix 기반)을 **필요한 것만 복사** — dialog, dropdown-menu, button, input, select, badge. 전체 킷 설치 금지.
- 아이콘: `lucide-react`. 토스트: `sonner`.
- 색: 뉴트럴(zinc) 스케일 + 액센트 1색(blue 계열). 카테고리 색은 8색 프리셋 팔레트(red/orange/amber/green/teal/blue/violet/pink)의 **키를 DB에 저장**하고(`categories.color`) CSS에서 매핑 — hex를 저장하지 않는다.
- 다크모드: `prefers-color-scheme` 기본 + 수동 토글(설정, localStorage 저장, `<html>` class 방식).
- 타이포: 시스템 폰트 스택 (`font-sans` 기본). 웹폰트 로딩 없음 (성능).
- 라운드/그림자: `rounded-xl`, 그림자 대신 `border` 위주 — 플랫하고 정돈된 인상.

## 라우트 구조 (TanStack Router 파일 기반)

```
src/routes/
├── __root.tsx          # html shell, QueryClientProvider, Toaster, 테마
├── login.tsx           # 로그인 (비인증 유일 화면)
└── _authed/            # beforeLoad에서 세션 가드 (04-auth)
    ├── route.tsx       # 레이아웃: 콘텐츠 + 하단 탭(모바일) / 상단 헤더(데스크톱)
    ├── index.tsx       # 홈: 북마크 목록
    ├── reminders.tsx   # 리마인더 목록
    └── settings.tsx    # 설정
```

네비게이션: 모바일은 하단 탭 3개(홈 / 리마인더 / 설정), `sm:` 이상은 상단 헤더로 전환.

## 화면 명세

### 홈 (`/`)

```
┌────────────────────────────────┐
│ My Bookmark            [＋ 추가]│  ← 헤더 (스크롤 시 고정)
│ [🔍 검색________________]      │  ← 입력 300ms debounce → q 파라미터
│ (전체) (미분류) (개발) (디자인)…  │  ← 카테고리 칩, 가로 스크롤, count 표시
├────────────────────────────────┤
│ ▢ favicon  제목                 │  ← BookmarkCard
│   domain · 카테고리배지 · 2일 전 │
│   [분석중…]                     │  ← ai_status=pending일 때 배지
│ ▢ …                            │
└────────────────────────────────┘
│  🏠 홈   ⏰ 리마인더   ⚙️ 설정   │  ← 하단 탭 (모바일)
```

- 카드 탭 → 원본 URL 새 탭 열기. 카드의 `⋯` 메뉴 → 카테고리 변경 / 리마인더 설정 / 편집 / 삭제 / (failed일 때) AI 재분류.
- 무한 스크롤: `useInfiniteQuery` + cursor. 100개 이상 렌더 시 TanStack Virtual 적용 (08-performance).
- `ai_status=pending` 항목이 있는 동안만 5초 간격 refetch로 분류 완료를 반영 (폴링은 pending 존재 시에만).
- 빈 상태: "첫 북마크를 추가해보세요" + 추가 버튼.

### 추가 다이얼로그 (홈의 [＋ 추가], 모바일에선 bottom sheet 스타일)

```
URL   [https://…________________]
분류   (● AI 자동)( 직접 선택 )( 미지정 )   ← 3-토글, 기본값: AI 자동
       └ "직접 선택" 시 카테고리 select 노출 (+ 새 카테고리 인라인 생성)
                              [저장]
```

- 저장 → optimistic하게 목록 반영, 409(중복)면 "이미 저장된 링크예요" 토스트 + 해당 카드로 스크롤.
- 마지막 선택 모드를 localStorage에 기억.

### 리마인더 (`/reminders`)

- pending 리마인더 목록 (시간순): 북마크 제목 + remind_at + note, 액션: 시간 변경 / 취소.
- 생성 진입점은 북마크 카드 메뉴의 "리마인더 설정" → 날짜/시간 피커 다이얼로그 (`datetime-local` 입력이면 충분).
- 알림이 꺼져 있으면 상단에 "푸시 알림이 꺼져 있어요 → 설정에서 켜기" 배너.

### 설정 (`/settings`)

섹션 순서:
1. **알림** — 푸시 켜기/끄기 토글, "테스트 알림 보내기" 버튼, iOS 미설치 시 안내문구 (06-pwa-push)
2. **카테고리 관리** — 목록(이름/색/개수), 추가·이름변경·색변경·삭제(확인 다이얼로그: "북마크 N개가 미분류가 됩니다")
3. **API 키** — 발급(이름 입력 → 키 1회 표시 + 복사 버튼 + "다시 볼 수 없어요" 경고), 목록(prefix, 마지막 사용), 회수. 하단에 단축어 설정 가이드 링크
4. **AI** — 모든 provider의 저비용/균형 모델을 그룹화한 단일 모델 선택기, 선택 모델 provider의 API 키 저장/교체, provider별 설정 여부·Models API 연결 테스트·키 삭제. 저장된 키 원문은 다시 표시하지 않음
5. **테마** — 시스템/라이트/다크
6. **계정** — 이메일 표시, 로그아웃

### 로그인 (`/login`)

중앙 카드 1개: 이메일/비밀번호/로그인 버튼. 실패 시 "이메일 또는 비밀번호가 올바르지 않습니다" (구체 사유 노출 금지). 가입/비밀번호 찾기 링크 없음 — 1인 서비스.

## 상태 관리 규칙

- 서버 상태는 전부 TanStack Query. 전역 클라이언트 상태 라이브러리(zustand 등) 도입 금지 — 테마/모드 기억 정도는 localStorage + React state로 충분.
- Query key: `['bookmarks', { categoryId, q }]`(infinite), `['categories']`, `['reminders']`, `['apiKeys']`.
- mutation 후 관련 key invalidate. 북마크 추가/삭제는 optimistic update 적용, 나머지는 invalidate만으로 충분.
- 폼: 추가 다이얼로그와 로그인은 TanStack Form + zod validator (`packages/shared` 스키마 재사용).

## 접근성/품질 기준

- 터치 타깃 최소 44px, 포커스 링 유지(Radix 기본), `aria-label` 필수인 아이콘 버튼.
- 모든 인터랙션에 로딩/에러 피드백 (버튼 spinner, sonner 토스트).
- 반응형 기준: 기본 = 모바일(~640px), `sm:` 이상 데스크톱 — 브레이크포인트 2단계만 사용.
